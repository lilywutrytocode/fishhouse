## 0. 文档约定

### 0.1 来源优先级
1. 主干 = 中泰《缠论技术分析框架解析》;2. 中泰模糊/缺失/冲突 → 缠师原文(REF);3. 两源冲突已逐条裁定,记录最终口径 + 出处。

### 0.2 三类标注
**【确定性】** 纯算法 / **【约定】** 已选定写死(部分标 `可配置`) / **【需判断】** 显式标 `未确认/待定/疑似`,交人工或 LLM,代码不假装能算。

### 0.3 构建顺序(自底向上)
数据层 → 包含 → 分型 → 笔 → 线段 → 中枢 → 背驰 → 三类买卖点 → 区间套/联立 → 概率/LLM → 输出/工程。每模块完成即跑该模块回归。

### 0.4 全局纪律
- **右端未确认**:最新 K 线附近结构默认 `未确认/待定`。
- **确定 vs 主观分层**:结构 + 监控位 + 联立结构信号 = 代码确定层(可复现);操作动作/LLM 复核 = 主观层,贴标,不伪装成算出来的。
- **数据门禁**:数据健康未过的标的×级别拒绝输出"已确认"结构。

### 0.5 ★ pivot vs confirm vs executable(防未来函数 —— 全局强制)
每个结构事件四个时间/价格锚:
- `pivot_date / pivot_price`:结构**极值**实际出现处。
- `confirm_date / confirm_price`:引擎**能确认**该结构成立处;**confirm_price 默认 = 确认 bar 的 close**。
- `executable_price`:**可交易模拟价**——日线信号默认**下一交易日 open**;30min 信号默认**下一根 30min bar open**。
- 若信号发生在最后一根 bar → 标 `live_pending`,**不纳入已完成回测**。
- **回测与实盘触发只准用 `confirm_date` + `executable_price`;严禁用 `pivot_*`。**

逐结构确认触发【确定性】:
| 结构 | pivot | confirm 触发 |
|---|---|---|
| 分型 | 中间 K 极值 | **第三根 K 收盘** |
| 笔 | 端点分型极值 K | 反向分型满足 4.1 成笔那刻(晚于端点分型 pivot) |
| 线段 | 端点极值 | 第一种:特征序列分型出现日;第二种:**第二特征序列分型出现日** |
| 中枢 | 首段起点 | 第三段成立、ZD≤ZG 那刻 |
| 一买 | 下跌极值低点 | 背驰确认 +(两态)次级别向上确认日 |
| 二买 | 回调低点 | 第一个次级别回调走势完成日(见 8.2) |
| 三买 | 回试低点 | 回试走势确认结束日(见 8.3) |
| 背驰 | 价格新高/新低点 | **C 段(笔/线段)确认完成**;C 段仍 extending → 只能 `提前判/疑似/待确认`,不得 confirmed |
> 卖点侧对称。tie-break(4.3)同价取最先**只影响 pivot 选择,不得提前 confirm_date**;confirm 仍按右侧确认规则。

### 0.6 ★ 通用结构 schema + 版本化(全结构强制)
每个结构对象含:`id, parent_id, source_unit_ids[], level, direction, status, pivot_date, pivot_price, confirm_date, confirm_price, executable_price`。
- id 规则:`{类型}_{级别}_{序号}`,如 `bi_d_017`、`xianduan_d_004`、`zhongshu_xd_d_002`、`beichi_d_009`、`signal_d_021`。
- 顶层输出含**版本化口径**:`spec_version(本期=v1.2), engine_version, data_snapshot_id, algorithm_config_hash`。
- 目的:同一标的不同时间跑出不同结果时,可区分"行情变了"还是"规格/代码/快照变了"。

## 模块 1 · 数据层

### 1.1 数据源【约定】
| 市场 | 标的 | 主源 | 兜底 |
|---|---|---|---|
| A 股 | 300502 / 300308 / 000792 | akshare | 东方财富 |
| 港股 | 09992 / 02400 | akshare | 东方财富 |
| 美股 | NVDA / SNDK / IONQ / SMMT | yfinance | — |
主源取不到前复权/数据不全 → 自动回落兜底。

### 1.2 复权【约定】
前复权钉死;**OHLC 四价全部前复权**(非仅 adjusted close)。**快照**(带拉取日)只读,保证可复现。

### 1.3 级别 + 降级隔离【约定】
- 日 + 周 + 30min。30min = 右端次级别确认器(美股仅近 ~60 天)。
- 30min 不可得历史段 → 降级"日线内部笔/线段近似确认",标 `次级别近似确认`。
- ★ 凡用日线内部近似的信号,**不进最高强度共振**,只标 `降级共振`(见 9.3/9.4)。

### 1.4 MACD【约定】12/26/9,收盘价。

### 1.5 数据长度【约定】日线 ≥ 250;周线 ≥ 150。

### 1.6 schema【约定】`date,open,high,low,close,volume,amount`;amount 可空。

### 1.7 数据健康【确定性】★(区分上市日/源起始/分析起点)
三个日期分开,**缺失率从 `analysis_start_date` 算**,不从源起始简单算:
- `listed_date`:上市/重新上市日。
- `source_start_date`:本次数据源可得最早日。
- `analysis_start_date`:实际用于结构计算的起点(取够 1.5 长度且数据质量达标处)。
- 输出 `pre_analysis_history_unavailable: bool`(源未给到上市日全历史时为 true,诚实标注)。

判定(在交易所交易日历内,假期不计):
- **缺失**:连续缺 ≥3 交易日 → REJECT;总缺失率(自 analysis_start_date)>5% → REJECT;否则 WARN(允许算,标注)。
- **历史充足度**:不足 1.5 → 不拒算,标 `SHORT_HISTORY`,拉满 + 告知,依赖长历史的判定(趋势背驰/多中枢/区间套)标 `低置信`。
- **停牌 ≠ 缺失**:A/港停牌日标 `SUSPENDED`,不计入缺失率、不触发 REJECT。
- 交易日历:A 股 akshare 自带;美股 `pandas_market_calendars`/`exchange_calendars`(NYSE);港股港交所日历。

### 1.8 交易制度/时间戳【约定】
时区统一带时区存储(沪/港/纽约),输出转用户本地;半日市按实际 session 生成 bar;美股**只用 regular session**(剔 pre/post)。

### 1.9 ★ 周线合成【约定】
周线**一律由已过健康检查的日线快照合成**(锚定周五 resample),**不直接读数据源周线**——保证复权口径与日线一致、可复现。

### 1.10 ★ 日内-日线一致性校验【确定性·可配置】
将 30min(regular session)聚合成日线 OHLCV,与日线源比较:
- close 偏差 > **0.5%** → WARN;high/low 偏差 > **1%** → WARN;任一核心价偏差 > **2%** → **REJECT 日-30min 联立**(禁止用此 30min 做日-30min 联立/共振,防口径不一致污染)。
- ★ 此 REJECT **仅作用于日-30min 联立**,**不 REJECT 单级别日线分析**——日线结构/背驰/买卖点仍照常输出。

## 模块 4 · 笔
> 出处:中泰 p25–26 / 缠师 65、66 课。**① 纯新笔(无缺口豁免)。**

**4.1 新笔判定**【确定性】① 顶底分型相邻;② 各自独立、不共用 K;③ 之间 ≥1 独立 K;④ 顶点>底点。→ 经包含处理通常 ≥5 根 K。
**4.2 方向有效性**【确定性】上升笔底点<顶点;下降反向。

### 4.3 笔划分算法【确定性·tie-break 已锁】(缠师 66 课三步)
1. 定分型:标出所有合标准分型。
2. **连续同类去重**:连续顶 → 取**价格最高**者;**若最高价相同,取最先者**。连续底 → 取**价格最低**者;同价取最先。
   - ★ 不得用"最先"代替"最高/最低"(二者仅去重后一致,以价格极值为准)。
   - ★ **同价 tie-break 只影响 pivot 选择,不得提前 confirm_date**;confirm 仍按成笔右侧确认。
3. 连接成笔:相邻顶底成一笔,不成笔即顺延至下一个满足 4.1 的反向分型。

**4.4 延伸/破坏**【确定性】笔只被笔破坏;新笔未成立算原笔延伸。右端未确认 → `笔·未确认/延伸`。

## 模块 5 · 线段
> 出处:中泰 p30(给原则)/ 缠师 67、71、78 课(特征序列法)。**乙 · 特征序列法。**
> 准确表述:历史已完成区间唯一划分;右端未确认/待定**必须显式输出**(不可称"全场景唯一确定")。

**5.1 特征序列构造**【约定+确定性】元素=反向笔,区间=[笔底点,笔顶点];缺口=相邻特征 K 无重叠。
**5.2 标准特征序列**【约定】仅同序列内包含处理、绝不跨转折点;方向=线段方向;向上线段只看顶分型、向下只看底分型。
**5.3 终结判定**【确定性】e₁=转折前最后特征元素,e₂=转折点后第一笔:第一种(e₁–e₂ 无缺口)出分型即终结;第二种(有缺口)需第二特征序列(严格包含处理)出反向分型才确认。
**5.4 成段前提**【确定性】≥3 笔、前三笔有重叠、顶>底、不被同向线段破坏、破坏逆时间传递;笔破坏前提:向上 ∃ j≥i+2 使 dⱼ≤gᵢ 且最后缺口封闭(向下对称)。"含笔数单数"仅作 assert/debug,不作筛选。
**5.5 标准化**【确定性】上层部件时极值标准化到端点。

### 5.6 右端未确认/待定【需判断·显式标注】
① 第二特征序列分型未出 → `未确认`;② 破坏笔是转折点第一笔且未延伸三笔 → `待定`;③ 第三笔完全落在第一笔内 → `待定`。

### 5.7 ★ 线段状态机【确定性】(核心,防右端跑偏)
状态枚举:`FORMING, EXTENDING, BREAK_CANDIDATE, WAIT_SECOND_FEATURE, PENDING_DIRECTION, CONFIRMED_END, INVALIDATED`。

| 当前状态 | 输入事件 | 转移条件 | → 转移到 | status 输出 | 允许喂中枢? |
|---|---|---|---|---|---|
| FORMING | 新笔 | 累计<3 笔 或 前三笔无重叠 | FORMING | 未确认 | 否 |
| FORMING | 新笔 | ≥3 笔 且 前三笔有重叠 | EXTENDING | 未确认 | 否 |
| EXTENDING | 新笔 | 同向延续、无反向特征序列分型 | EXTENDING | 未确认·延伸 | 否 |
| EXTENDING | 新笔 | 反向特征序列出现分型(满足 5.4 笔破坏前提) | BREAK_CANDIDATE | 未确认 | 否 |
| EXTENDING | 新笔 | 第三笔落在第一笔内/方向未定 | PENDING_DIRECTION | 待定 | 否 |
| BREAK_CANDIDATE | 判定 | e₁–e₂ **无缺口**(第一种) | CONFIRMED_END | 已确认 | — |
| BREAK_CANDIDATE | 判定 | e₁–e₂ **有缺口**(第二种) | WAIT_SECOND_FEATURE | 未确认 | 否 |
| WAIT_SECOND_FEATURE | 新笔 | 第二特征序列出现反向分型 | CONFIRMED_END | 已确认 | — |
| WAIT_SECOND_FEATURE | 新笔 | 原方向重新创新高/低(候选作废) | INVALIDATED | 未确认 | 否 |
| PENDING_DIRECTION | 新笔 | 先破终结侧 | CONFIRMED_END | 已确认 | — |
| PENDING_DIRECTION | 新笔 | 先破延续侧 | EXTENDING | 未确认·延伸 | 否 |
| CONFIRMED_END | — | confirm_date=终结确认日;新线段自终结点起 | FORMING(新) | 已确认 | **是** |
| INVALIDATED | — | 候选作废,并回原线段延伸 | EXTENDING | 未确认 | 否 |

★ **只有 CONFIRMED_END 产出的已完成线段允许喂线段中枢**;右端非 CONFIRMED_END 的线段标未确认,不入确定性中枢(可作"未确认中枢"候选)。

## 模块 6 · 中枢
> 出处:中泰 p39/p44/p46 / 缠师 17 课 + 中心定理一二三。**笔中枢 + 线段中枢都做。**

### 6.1 ZG/ZD/GG/DD【确定性】
成立时(连续三个次级别走势单位 U1/U2/U3):
```
ZG = min(high(U1),high(U2),high(U3))   # 重叠区间上沿
ZD = max(low(U1), low(U2), low(U3))    # 重叠区间下沿
ZD ≤ ZG → 形成中枢,区间 [ZD,ZG]
```
成立后 ZG/ZD 固定;`GG=max(各延伸同向段高)`、`DD=min(各延伸同向段低)` 随延伸刷新。

**6.2 成立**【确定性】ZD ≤ ZG。
**6.3 延伸**【确定性·定理一】任一同向段 [dₙ,gₙ] 与 [ZD,ZG] 有重叠则延伸;dₙ>ZG 或 gₙ<ZD 则转高级别/趋势。不设硬上限,段数标注。
**6.4 扩展**【确定性·定理二】后 GG<前 DD → 下跌延续;后 DD>前 GG → 上涨延续;后 ZG<前 ZD 且 后 GG≥前 DD(或对称)→ 高级别中枢。
**6.5 破坏**【确定性·定理三】次级别离开后回抽不回 [ZD,ZG] 内 → 旧中枢破坏 → 新中枢/趋势。
**6.6 起点**【约定·①】标准起点,不做重新组合优化。

### 6.7 ★ 信号去重 / 事件簇【约定·可配置】(笔+线段中枢都做的必备)
**同一事件簇**条件(全满足):同标的、**同级别、同方向、同类买卖点**;`confirm_date 相差 ≤ 3 根K线`;`confirm_price 距离 ≤ 1%`。(N=3、x=1% 可配置)
- ★ **只在同级别内去重**:跨级别信号(如日线三买 vs 周线三买)是**不同事件,都保留**,绝不合并。
事件簇内**保留优先级**(均为同级别内的区分):线段中枢 > 笔中枢;标准背驰 > 盘背;真 30min 确认 > 日线内部近似;**confirm_date 更早者为触发**,其余进 `supporting_signals[]`(保留 id,不重复计入回测)。

## 模块 7 · 背驰
> 出处:中泰 p58/p6 / 缠师 24、27 课。

### 7.1 动能度量【确定性】段内 MACD 柱面积(向上红柱、向下绿柱累计)。

### 7.2 分档输出 + C 段可测【约定】★
前提:价格创新高/新低。两条件各按 7.3 三态评估,按"满足哪些"分三档:
- **标准背驰** = 面积↓ 且 DIF 峰值↓ → **主信号**(仓位/共振只用此档)。
- **面积背驰(弱)** = 仅面积↓;**DIF 背驰(弱)** = 仅 DIF↓ → 进报告 + 回测,非主信号。
- ★ **C 段完成可测**:背驰比较单元为笔级 → C 段完成 = C 笔 confirmed;线段级 → C 段完成 = C 线段 CONFIRMED_END。**C 段仍 extending/pending 时,只能 `beichi_status ∈ {提前判, 疑似, 待确认}`,不得输出 confirmed 背驰**(防"面积已经小了"被直接当 confirmed)。

### 7.3 容差三态【约定·k=0.9】对每个条件(面积/DIF 峰值),C/A:<0.9 满足;[0.9,1.0] 疑似;>1.0 不满足。

### 7.4 趋势 vs 盘整背驰【确定性·分别标注】
- **趋势背驰**:≥2 同级别中枢;中枢 B 隔开的同向 A(进入)vs C(离开);C 创新高/新低且 C 面积<A(且 DIF↓)。
  - ★ "黄白线回拉 0 轴附近" **v1 不作硬门槛**,只输出字段 `macd_reset_status ∈ {已回拉0轴, 接近0轴, 未回拉}`(避免肉眼阈值成分歧源;日后回测再定是否硬约束)。
- **盘整背驰**:同一中枢内,★ **按"最近一组同向 A/C"识别**——A = 本次进入中枢前的同向段,C = 本次离开/试图离开中枢的同向段;**中枢延伸超 3 段时不固定取前三段**,以免错位。

**7.5 比较单元**【确定性】跟随中枢单位(线段中枢→线段级;笔中枢→笔级),都算、各标级别。

## 模块 8 · 三类买卖点
> 出处:中泰 p11/p59/p60/p62 / 缠师 14、21、67 课 + 第三类定理。

### 8.1 一买【确定性】★(标准/盘背 + 中枢位置双字段)
某级别下跌走势完成:
- **趋势背驰后 → `一买·标准`**;**盘整背驰后 → `一买·盘背`(类一买)**,仓位不同等(缠师 67 课)。
- 输出两个位置字段:`pivot_relation_to_zhongshu` 与 `confirm_relation_to_zhongshu` ∈ {below, inside, above}(相对最近同级别中枢)。**结构成立看 pivot(应 below);操作风险看 confirm**(确认时可能已回到 inside,操作含义不同)。

### 8.2 二买【确定性】★(第一个次级别回调识别 5 步,14 课)
一买 confirmed 后:① 等一个次级别**向上**走势确认 → ② 等其后**第一个**次级别**向下**走势完成 → ③ 该向下走势低点 **> 一买 pivot_price** → ④ 该低点 = 二买 pivot → ⑤ 该向下走势完成日 = 二买 confirm。防止把一买后第一个小低点/段内波动误判成二买。二卖对称。

### 8.3 三买【确定性】★(离开/回试最小条件 + 引用)
次级别向上离开中枢 + 回试低点 > ZG。防误判:
- **离开**:必须至少形成**一个已确认次级别走势单位**,其 low/high 明确脱离中枢,**不能只用单根 K 刺破 ZG**。
- **回试**:回试走势本身必须**确认结束**,不得在 extending 时输出 confirmed 三买。
- 记录 `leave_unit_id`、`retest_unit_id`。三卖对称(回抽高点 < ZD)。

**8.4 次级别确认(一买)**【约定·①】日线背驰→`一买·背驰确认`;次级别向上→`一买·次级别确认`;历史段降级近似 + 标注;右端未出→`待确认`。
**8.5 级别**【确定性】= 背驰/中枢级别。
**8.6 重合**【确定性】二买与三买可重合→`二三买重合`;一买/二买、一买/三买不重合。
**8.7 剔除项**【非主干】中泰 p63 均线/量能默认不纳入,作可选叠加层贴标。

**9.2 区间套定位**【确定性】嵌套背驰段 [周]⊃[日]⊃[30min] 逐级收缩,精确点落最低可得级别背驰段内。

### 9.3 联立结构信号【确定性】★(仅标准背驰参与共振 + 降级隔离)
| 周 | 日 | 30min | 结构信号 |
|---|---|---|---|
| 标准背驰 | 标准背驰 | 标准背驰(真 30min) | **共振背驰**(最强转折) |
| 标准背驰 | 标准背驰 | 标准背驰(日线内部近似) | **降级共振/近似共振**(强度降一档) |
| 无 | 背驰 | 背驰/确认 | 本级别转折成立 |
| 无 | 无 | 背驰 | 次级别回调·保护级(非反转) |
| 任意 | 背驰后未反向、盘整顺原向 | — | 小转大/信号失效 |
★ 凡 30min 用日线内部近似,一律不进"共振背驰"最高强度,只进"降级共振"。

### 9.4 操作映射【需判断·主观·可配置】★(降级共振口径已锁)
引擎核心只出 {结构信号 + 级别 + 共振状态(含是否降级) + 区间套区间};**不出"算出来的"仓位数字**,仓位只给强度档:
| 结构信号 | 默认动作(主观·可配置) |
|---|---|
| **真共振背驰** | 最高强度:顶 → 清仓/大幅减;底 → 重仓建仓 |
| **降级共振/近似共振** | **降一档**:顶 → 减仓/降风险(**不清仓**);底 → 试仓/分批建仓(**不重仓一次性**)。若后续真 30min 确认补齐 → 可升级为真共振 |
| 本级别转折(次级别确认) | 减仓 · 建仓 |
| 次级别回调·保护级 | 减仓/对冲,非清仓(缠师 REF p155) |
| 小转大/失效 | 撤销减仓信号,持有/顺势 |

### 9.5 盲复核旁路【主观·非确定·非复现】
LLM 不见 policy,只收 {结构输出 + 数值快照};用同一动作词汇出 `{加/持/减/清+强度+理由}`;引擎比对标 `policy_divergence`。★ **LLM 输出只进 `review_notes`,不得写结构确定字段、不得把未确认改 confirmed**。

### 11.5 测试【约定·③ 两者 + 6 类极端 + 未来函数断言】
